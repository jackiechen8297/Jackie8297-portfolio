Executive Summary

An API (Application Programming Interface) is a type of software interface that allows for the exchange of data, code, and other components between the users and various software applications, servers, systems. This study section reports on the implementation of a Plant identifier API that is capable of identifying sick plants when a photo of a plant is inserted into the API. This API returns the scientific name, common name, disease probability, type of disease, and other components that the API returns. This plant disease identifier API can be used to help people who work in the agricultural sector identify plant diseases in the event of their plants becoming unhealthy. Ideally, plant identifier API can be a viable solution method to prevent the spread of plant diseases and crop failure due to the disease identification component of this API.

An understanding of the JavaScript code should also be taken into consideration in order to create the functions that will identify the plant disease.

Project Analysis

Contributing Factors to the Issue: The main factor contributing to the issue would be the lack of information and knowledge on the common plant diseases. The pace of identifying the plant disease is also another factor to take into consideration.

Strengths and Weaknesses of the API: The strength of the API lies in its efficiency of identifying the plant health issue due to its quick access to the plant information database.

Steps Taken: The steps that were taken weas gradually easing into the concepts aand frameworks of JSON (JavaScript Object Notation), Axios (JavaScript Library), APIs, Postman(platform for API building)

Concepts learned from API: Familiarization of several Javascript frameworks & libaries, HTTP methods (GET, POST, PUT, DELETE), and debugging/troubleshooting null errors were some of the concepts that were picked up during the API integration process.

Challenges

The challenges included many bugs that were present in the Javascript functions and parameters. There were many syntax errors, unexpected datatypes, null properties, and other case sensitivity issues. Some of the functions did not work due to missing semi-colons or backticks.

Solution

Debugging and troubleshooting errors, and double checking if the HTML code contained the same spelling and case sensitivity as the JavaScript code. Many of the spelling mistakes and case sensivity prevented the API from functioning properly. The API was finally able to function properly when all the spellings were corrected.

Conclusion

In the end, the API was able to succesfully identify the plant and the disease associated with the plant when the image is inserted into the API. In the future, I intend to further develop this project by incorporating additional identification features, which will include a design with the intent to enhance the accuracy of plant recognition and expand the categorization of species.
